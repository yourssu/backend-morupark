# 인프라 디렉토리 안내

`infra/`는 운영 인프라 관련 리소스를 목적별로 분리해 관리합니다.

## 디렉토리 역할

- `terraform/`
  - GCP/Cloudflare 등 인프라를 코드로 관리하는 Terraform 루트입니다.
  - `modules/`에는 재사용 가능한 모듈을, `environments/`에는 환경별 조합(예: `prod`)을 둡니다.

- `k8s/`
  - Kubernetes 매니페스트를 관리합니다.
  - 공통 `base`, 서비스별 `components`, 환경별 `overlays`, 배포 연계용 `argocd` 구성을 포함합니다.

- `scripts/`
  - 반복 운영 작업을 자동화한 스크립트 모음입니다.
  - `deploy/`(재배포/배포 보조), `setup/`(초기 준비), `utility/`(점검/유틸)로 나눕니다.

- `load-test/`
  - k6 기반 부하/스트레스 테스트 자산을 보관합니다.
  - 테스트 시나리오와 실행 가이드를 함께 관리합니다.

- `legacy/`
  - 현재 표준 경로로 이관되었거나 사용 빈도가 낮은 과거 자산을 임시 보관하는 영역입니다.
  - 신규 작업은 가급적 `terraform/`, `k8s/`, `scripts/`, `load-test/`를 우선 사용합니다.

## 바이너리 관리 정책

`kubectl.exe` 같은 플랫폼 바이너리는 저장소에 커밋하지 않습니다.
대신 문서화된 버전 고정 및 설치 절차를 사용합니다.
