# skill-ssl-authoring 가이드

이 디렉토리는 새로운 로컬 스킬을 만들거나, 기존 텍스트 중심 스킬을 `SKILL.md + ssl.json` 구조로 정리할 때 사용하는 스킬 패키지입니다.
즉, "스킬을 만드는 방법을 안내하는 스킬"이라고 보면 됩니다.

## 이 디렉토리의 역할

- 저장소 안의 스킬 구조를 표준화합니다.
- 사람이 읽는 안내서와 기계가 읽는 구조화 그래프를 분리하도록 유도합니다.
- 새 스킬 작성, 기존 스킬 변환, 구조 검증을 돕습니다.

## 파일과 디렉토리 설명

- `SKILL.md`
  - 이 스킬 자체의 간단한 사용 설명서입니다.
  - 어떤 파일들을 결과물로 만들어야 하는지와 작성 원칙이 요약되어 있습니다.
- `ssl.json`
  - 스킬 작성 과정을 구조화한 파일입니다.
  - 소스 수집, 스케줄링 추출, 구조 분해, 논리 단계 작성, 검증 흐름을 담습니다.
- `agents/`
  - 이 스킬을 UI에서 어떻게 보여줄지 정의하는 메타데이터가 있습니다.
- `references/`
  - 스킬 변환 절차와 `ssl.json` 스키마 설명을 담는 참조 문서 폴더입니다.
- `assets/`
  - 새 `ssl.json`을 만들 때 시작점으로 쓸 수 있는 템플릿 파일을 둡니다.

## 어떻게 사용하면 되나

1. 새 스킬을 만들거나 기존 스킬을 정리하려면 먼저 `SKILL.md`를 읽습니다.
2. 구체적인 작성 방식이 필요하면 `references/` 문서를 봅니다.
3. 실제 `ssl.json` 초안을 만들 때는 `assets/ssl-template.json`을 출발점으로 사용하면 편합니다.
4. 완성 후에는 `ssl.json`의 scene, step, grounding 연결이 맞는지 검토합니다.

## 주의할 점

- 이 스킬의 목적은 "문서를 길게 쓰는 것"이 아니라 역할을 분리해 관리 가능하게 만드는 것입니다.
- `SKILL.md`에 모든 내용을 몰아넣기보다, 개요와 상세 구조를 적절히 나누는 편이 좋습니다.
