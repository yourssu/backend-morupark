# .agents/skills 가이드

이 디렉토리는 저장소에서 사용할 수 있는 로컬 스킬들의 모음입니다.
각 하위 디렉토리는 하나의 스킬 패키지이며, 특정 작업 영역을 담당합니다.

## 이 디렉토리의 역할

- 에이전트가 호출할 수 있는 작업 단위를 스킬별로 분리합니다.
- 스킬마다 설명, 구조화 데이터, 참조 문서, 메타데이터를 함께 둡니다.

## 포함된 스킬 디렉토리

- `harness/`
  - 운영 요청을 적절한 하위 스킬로 연결해 주는 메타 스킬입니다.
- `eso-recurrence-prevention/`
  - ESO, CRD, Secret 동기화, 배포 순서 문제의 재발을 막는 전용 스킬입니다.
- `cluster-reprovision-handoff/`
  - 클러스터 재구축이나 Terraform 이후 인프라 인계 절차를 다루는 스킬입니다.
- `k8s-release-verification/`
  - Kubernetes 배포 후 상태 확인과 롤아웃 문제 분류를 다루는 스킬입니다.
- `infra-ops/`
  - 이 저장소의 `infra/` 관련 운영 작업을 다루는 스킬입니다.
- `skill-ssl-authoring/`
  - 새로운 스킬을 `SKILL.md + ssl.json` 구조로 만드는 방법을 다루는 스킬입니다.

## 보통의 스킬 구성

- `SKILL.md`
  - 사람이 먼저 읽는 핵심 안내서입니다.
- `ssl.json`
  - 스킬의 실행 흐름을 구조화한 파일입니다.
- `agents/`
  - 에이전트 UI나 디스커버리용 메타데이터를 둡니다.
- `references/`
  - 긴 설명, 절차, 세부 규칙을 분리해 둡니다.
- `assets/`
  - 템플릿이나 재사용 가능한 정적 자료를 둡니다.

## 어떻게 사용하면 되나

1. 하려는 작업과 맞는 스킬 디렉토리를 고릅니다.
2. 먼저 `SKILL.md`를 읽고 용도와 절차를 이해합니다.
3. 필요하면 `references/`에서 세부 설명을 확인합니다.
4. 스킬 자체를 수정하거나 자동화하려면 `ssl.json`도 함께 봅니다.

## 새 스킬을 추가할 때

- 기존 디렉토리 구조를 복사해 시작하는 것이 가장 안전합니다.
- 단순 설명만 필요한 경우에도 최소한 `SKILL.md`와 `ssl.json` 역할 분리는 유지하는 것이 좋습니다.
