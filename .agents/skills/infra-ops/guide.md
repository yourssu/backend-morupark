# infra-ops 가이드

이 디렉토리는 MoruPark 저장소의 인프라 작업을 안전하게 다루기 위한 스킬 패키지입니다.
주로 Terraform, GCP, GKE, Cloud SQL, Secret Manager, Cloudflare, Kubernetes 연계 작업을 대상으로 합니다.

## 이 디렉토리의 역할

- `infra/` 아래 운영 작업의 표준 절차를 정의합니다.
- 변경 전 점검, `plan`, 적용 후 검증, 후속 작업까지 한 흐름으로 안내합니다.
- 특히 운영 환경(`prod`)에서 실수하지 않도록 가드레일을 제공합니다.

## 파일과 디렉토리 설명

- `SKILL.md`
  - 사람이 읽는 핵심 사용 가이드입니다.
  - 언제 이 스킬을 써야 하는지, 어떤 순서로 점검해야 하는지 요약합니다.
- `ssl.json`
  - 스킬의 실행 흐름을 구조화한 파일입니다.
  - 어떤 입력이 필요하고, 어떤 단계가 있고, 어디서 위험이 생기는지 기계적으로 표현합니다.
- `agents/`
  - 이 스킬을 UI나 에이전트 목록에서 어떻게 보여줄지 정의하는 메타데이터가 있습니다.
- `references/`
  - Terraform 절차와 후속 운영 작업처럼 길어질 수 있는 설명을 분리해 둔 폴더입니다.

## 어떻게 사용하면 되나

1. 인프라 관련 요청이 들어오면 먼저 `SKILL.md`를 읽습니다.
2. Terraform 작업이면 `references/terraform-workflow.md`를 함께 봅니다.
3. 적용 이후 DB 생성, 시크릿, DNS, K8s 인계가 필요하면 `references/ops-followups.md`를 확인합니다.
4. 스킬을 자동화하거나 구조적으로 검토해야 하면 `ssl.json`을 읽습니다.

## 주의할 점

- 이 스킬은 운영 환경 변경 가능성이 있는 작업을 다룹니다.
- `apply` 같은 실제 변경 작업은 항상 명시적 승인과 사전 검토가 필요하다는 전제를 갖고 있습니다.
- `SKILL.md`는 짧게, 세부 구조는 `ssl.json`과 `references/`에 두는 방식으로 유지하는 편이 좋습니다.
